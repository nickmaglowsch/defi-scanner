# Task 10: Historical Percentile and Rank Computation

- **Decisions**: `get_historical_rank` calls `get_percentile` internally, which means the route makes 2 `execute` calls for percentile data (one for `get_percentile`, one when `get_historical_rank` calls it again). This doubles the DB round-trips vs. computing both in one query. Chose this for separation of concerns — the functions are independently useful. Upgrade path: add a combined `get_percentile_and_rank` that does one query.
- **Decisions**: The percentile SQL uses `PERCENT_RANK()` window function. This correctly handles ties (tied values get the same percentile). `CUME_DIST()` was the alternative — chose `PERCENT_RANK()` per task spec.
- **Decisions**: Thresholds for `_percentile_to_rank_label`: top 5%, top 10%, top 25%, above/below median, bottom 25%. These are intuitive enough for a first version; no prior spec, so I chose common quantile labels.
- **Deviations**: The `_MIN_DAYS` check uses distinct calendar days from `observed_at`. The SQL query uses `PERCENT_RANK()` without a separate day-count query — instead it relies on the `counts` CTE which counts distinct days in the same SQL block. This is more efficient than two queries.
- **Deviations**: `get_percentile` returns `None` (not `{}`) for markets that appear in the query result but have insufficient data. Markets with no rows at all also get `None` via the "fill missing" loop at the end.
- **Risks**: The SQL interval workaround (`:window_days` replaced via string formatting) is safe because `window_days` is an internal `int`, not user input. This bypasses parameterization for the interval — acceptable but document it clearly (done via ponytail comment).
- **Risks**: Existing `test_api.py` mock lists required surgery — adding 2 mocks per loop/carry mock helper (`percentile_result`, `rank_percentile_result`). Also had to update the `mocks[6]` → `mocks[8]` index in `test_opportunities_sharpe_null_on_zero_volatility`. If more queries are added in future, this index will need updating again — a fragile pattern.
