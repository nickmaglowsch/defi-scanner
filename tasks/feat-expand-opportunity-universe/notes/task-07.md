# Task 07: Collector Runner Concurrency Refactor

- **Decisions**: Extracted `_build_lending_collector`, `_build_funding_collector`, and `_start_infrastructure_tasks` as module-level functions (not methods or classes) so tests can `patch.object` them without touching the registry or real adapter constructors. This is the minimal patchable surface.
- **Decisions**: Per-chain RPC URLs (`DEFI_RPC_URL_BASE`, etc.) are read from `os.environ` at import time rather than added to `Settings`, because adding every possible chain as a typed field would require knowing all chains upfront. The `Settings` env_prefix `DEFI_` would require the user to pass `DEFI_DEFI_RPC_URL_BASE` if we used Settings, so direct `os.environ` reads use the natural key names.
- **Decisions**: `shutdown_collectors()` now also cancels tasks directly (in addition to setting the event) so callers don't need to wait for the event-driven cancellation path inside `run_collectors`. This fixes the test where `run_collectors` isn't running.
- **Decisions**: `RegistryEntry.enabled` defaults to `True` so all existing YAML entries (which lack the field) are automatically enabled without a migration.
- **Deviations**: The `_shutdown_event` is module-level and loop-bound. Tests must assign `runner_module._shutdown_event = asyncio.Event()` (not `.clear()`) before each test to avoid "bound to a different event loop" errors under pytest-asyncio strict mode. This is documented via the comment in the test.
- **Trade-offs**: Considered using an importlib-based lazy registry instead of a slug `if/elif` chain. Rejected: the slug list is short, the `if/elif` is readable at a glance, and any new adapter needs an explicit entry anyway (no magic autoload).
- **Risks**: Stub adapters (GMX, Drift, Vertex, dYdX) raise `NotImplementedError` at fetch time. The `_run_loop` exception handler catches this and retries every `COLLECTOR_INTERVAL_SECONDS`. This produces log noise but is harmless. Remove the entries from the registry or add `enabled: false` to silence them.
