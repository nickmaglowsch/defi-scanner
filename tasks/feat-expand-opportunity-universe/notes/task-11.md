# Task 11: New Strategy Providers (Stable Lending, Staking, Restaking, Pendle)

- **Decisions**: `StableLendingAdapter` returns an empty list (no `NotImplementedError`) because stable lending is derivable from existing lending snapshots — the task says it "can be synthetically derived." Returning empty is honest and doesn't break callers. The route's `_fetch_market_type_opportunities` would find snapshots if any collectors with `market_type="stable_lending"` were wired in.
- **Decisions**: `StakingAdapter` raises `NotImplementedError` rather than returning empty. The task says "Lido/Rocket Pool APIs or on-chain exchange rates" — these require live HTTP access that test isolation doesn't have. The error message describes the upgrade path (Lido APR endpoint).
- **Decisions**: `RestakingAdapter` and `StakingAdapter` both raise `NotImplementedError`. EigenLayer AVS reward data has no stable public REST API as of 2026 (confirmed by task notes). The error message documents the contract addresses for the upgrade path.
- **Decisions**: `PendleAdapter` conditionally raises `NotImplementedError` when no client is injected, and attempts the API when a client is provided. This matches the Morpho adapter pattern (owns-client pattern). The mocked API test validates the normalization logic independently of network access.
- **Decisions**: `_fetch_market_type_opportunities` in routes serves all four new types from `lending_snapshots` filtered by `market_type`. This is intentional — these adapters would write to `lending_snapshots` (same table as lending) when wired in, with distinct `market_type` values. No new tables needed.
- **Decisions**: `trigger_loop_calculation` in orchestrator now checks `market.market_type in _NO_BORROW_LEG_MARKET_TYPES` and skips. This is the minimal change — one guard in the shared function vs. separate checks in every caller.
- **Trade-offs**: Could have wired stable_lending into `type=all` results by filtering lending snapshots with `borrow_apy=None`. Skipped — that would incorrectly capture lending markets where borrow just wasn't reported yet. Requires an explicit `market_type` to distinguish.
- **Risks**: The Pendle test `test_with_mocked_api_response` only validates shape, not that specific fields like `fixedApy` are correctly parsed. If the Pendle API changes its response schema, the test would still pass but production data would be wrong.
